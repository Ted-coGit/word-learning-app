## Version 0.2 (2025-01-25)

### 개발 배경
- 멀티 디바이스 환경에서 단어장 동기화 필요성 발생
- 호경님이 "선생님" 역할로 아이들에게 단어 배포하는 시나리오
- GitHub를 원본 소스로 활용한 중앙 관리 시스템 구현

### 핵심 요구사항
1. **GitHub 기반 중앙 관리**: CSV 파일을 GitHub에서 관리
2. **멀티 디바이스 지원**: 태블릿, 아이폰 등 여러 기기에서 동일한 최신 단어 접근
3. **원클릭 업데이트**: 새로고침 버튼으로 최신 단어 자동 다운로드
4. **선택적 업데이트**: 기존 단어에 추가 vs 완전 교체

### 추가 기능

#### 1. GitHub CSV 자동 불러오기
- GitHub repository의 `data/` 폴더에서 사용자별 CSV 파일 다운로드
- 파일 경로: `https://raw.githubusercontent.com/Ted-coGit/word-learning-app/main/data/[사용자명].csv`
- 중복 단어 자동 스킵 (추가 모드 시)

#### 2. 새로고침 버튼 (🔄)
- 메인 메뉴 상단에 "🔄 새로고침" 버튼 추가
- 클릭 시 모드 선택 팝업:
  - **확인**: 기존 단어에 추가 (중복 제외)
  - **취소**: 완전히 새로 시작 (기존 단어 삭제)

#### 3. 데이터 소스 이중화
```
로컬 데이터 (localStorage)
    ↕️ 양방향
GitHub 데이터 (중앙 저장소)
```

### 사용 시나리오

#### 시나리오 1: 선생님 → 학생 배포
```
[밤 - 호경님]
1. 엑셀에서 이번 주 단어 5개 작성
2. "진성운.csv"로 저장
3. GitHub data/ 폴더에 업로드

[아침 - 진성운]
1. 태블릿에서 앱 실행
2. "진성운" 선택
3. 🔄 새로고침 클릭
4. 새 단어 자동 추가
5. 학습 시작!
```

#### 시나리오 2: 멀티 디바이스
```
[진성운 태블릿]
단어 학습 중

[외출 - 호경님 아이폰]
"진성운" 선택 → 새로고침
→ 태블릿과 동일한 최신 단어로 학습
```

#### 시나리오 3: 주간 업데이트
```
[주말 - 호경님]
1. 이번 주 틀린 단어 정리
2. 복습 단어 추가한 CSV 생성
3. GitHub 업로드
4. 아이들이 새로고침하면 자동 업데이트
```

### 기술 구현

#### GitHub Raw Content API
```javascript
const url = `https://raw.githubusercontent.com/${username}/${repo}/main/data/${filename}`;
const response = await fetch(url);
const csvText = await response.text();
```

#### 중복 체크 로직
```javascript
const isDuplicate = existingWords.some(
  w => w.english.toLowerCase() === newWord.english.toLowerCase()
);
if (isDuplicate && mode === 'add') continue;
```

#### 모드 선택 UI
```javascript
const mode = window.confirm(
  '확인 = 기존 단어에 추가\n' +
  '취소 = 완전히 새로 시작'
);
loadFromGitHub(mode ? 'add' : 'replace');
```

### 장점

✅ **중앙 집중식 관리**: 한 곳(GitHub)에서 모든 단어 관리  
✅ **멀티 디바이스 지원**: 어떤 기기에서든 동일한 최신 단어  
✅ **자동 백업**: GitHub에 영구 보관  
✅ **버전 관리**: GitHub의 commit 히스토리로 변경 추적  
✅ **오프라인 학습**: 한 번 다운로드하면 오프라인 가능  

### 제한사항

⚠️ **인터넷 연결 필요**: 새로고침 시에만  
⚠️ **수동 업데이트**: 자동 푸시 알림 없음 (아이들이 새로고침 버튼 클릭 필요)  
⚠️ **Public Repository**: GitHub URL이 공개되어 있음 (개인정보 없음)  

### 보안 고려사항

- CSV 파일은 Public repository에 저장되므로 개인정보 포함 금지
- 단어와 뜻만 저장 (이름, 연락처 등 개인정보 없음)
- 학습 기록은 각 기기의 localStorage에만 저장

## Version 0.1 (2025-01-25)

### 개발 배경
- 초등학교 3학년 아들(진성운)의 영어 단어 학습을 위해 제작
- 알파벳을 가끔 헷갈리는 수준의 초보 학습자 고려
- 2일에 한 번 꼴로 아빠와 함께 학습하는 시나리오

### 핵심 요구사항
1. **학습 대상**: 초등학교 3학년, 영어 초보 수준
2. **학습 목표**: 단어 스펠링 암기
3. **학습 방법**: 짝맞추기 + 스펠링 쓰기
4. **피드백**: 정답 시 발음 + 칭찬 효과음, 오답 시 재미있는 효과음
5. **학습 주기**: 신규 학습 + 복습의 2회 세션 구조
6. **다중 사용자**: 3명(아빠, 큰아들, 막내아들) 독립 학습

### 주요 기능

#### 1. 사용자 프로필 시스템
- 3명의 사용자 독립 관리
  - 👨‍💼 진호경 (파란색)
  - 👦 진성운 (초록색)
  - 🧒 진성율 (주황색)
- 사용자별 독립적인 단어장 데이터
- 사용자 전환 기능

#### 2. 단어 관리
- **이번 주 단어**: 현재 학습 중인 신규 단어
- **복습 단어**: 학습 완료 후 반복 학습용 단어
- 단어 추가/삭제 기능
- 발음 듣기 (Web Speech API)
- 단어장 간 이동 (이번 주 → 복습)

#### 3. 학습 모드

**짝맞추기**
- 한글 뜻 보고 4개 선택지에서 영어 단어 선택
- 최소 4개 단어 필요
- 틀리면 재도전 가능
- 넘어가기 버튼으로 어려운 단어 스킵

**스펠링 쓰기**
- 한글 뜻 보고 직접 영어 단어 입력
- 대소문자 구분 없음
- 발음 듣기 지원
- 틀리면 입력창 초기화 후 재도전
- 넘어가기 버튼으로 어려운 단어 스킵

#### 4. 피드백 시스템
- **정답**: 
  - 상승하는 화음 효과음
  - 단어 발음 재생
  - "🎉 잘했어요!", "🌟 완벽해요!" 메시지
- **오답**:
  - 하강하는 재미있는 효과음
  - "😊 다시 해볼까요!" 메시지
  - 재도전 또는 넘어가기 선택 가능

#### 5. 결과 화면
- 점수 및 정답률 표시
- 정답률별 격려 메시지
  - 100%: "🏆 완벽해요! 천재예요!"
  - 80%+: "🎖️ 정말 잘했어요!"
  - 60%+: "👍 좋아요! 조금만 더!"
  - 60% 미만: "💪 괜찮아요! 다시 해봐요!"

#### 6. CSV 내보내기/불러오기
- **내보내기**:
  - 파일명 형식: `단어장_[사용자명]_[날짜].csv`
  - 모든 단어(이번 주 + 복습) 포함
  - 엑셀에서 편집 가능
- **불러오기**:
  - CSV 파일 업로드
  - 기존 단어에 추가하는 방식
  - 대량 단어 입력 가능

**Issue 3: 다중 사용자 관리**
- 문제: 여러 명이 사용할 때 데이터 분리 필요
- 해결: 사용자별 localStorage 키 분리
